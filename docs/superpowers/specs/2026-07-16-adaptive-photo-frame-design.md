# 旅行照片方向自适应相框设计

## 问题

当前相框宽度随页面变化、高度封顶 500px，导致同一张竖图在手机上接近竖框、在宽屏上却变成横框，裁切结果不一致。

## 决策

- 每个旅行站点显式声明照片方向：`portrait` 或 `landscape`。
- 竖图相框固定为 3:4，横图相框固定为 4:3。
- 相框宽度与高度同时受当前视口约束：不会因宽屏无限变宽，也不会在矮屏挤掉文案和按钮。
- 图片继续使用 `object-fit: cover`，允许通过 `objectPosition` 微调主体；不拉伸、不添加滤镜。
- 未替换的示例站点暂按横图处理，替换真实照片时根据实际方向更新。
- 芽庄照片为 1279×1706，声明为 `portrait`。

## 验收

- 芽庄站点渲染竖向 3:4 相框。
- 横向站点渲染横向 4:3 相框。
- 390×844 与 760×900 视口中，相框方向一致。
- 人物、椰子和背景招牌在芽庄照片中保持可辨认，不因宽屏改为横向裁切。
- 自动化测试、生产构建与 GitHub Pages 部署通过。
